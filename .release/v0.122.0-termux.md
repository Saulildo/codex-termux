# Termux fork notes

This release rebases the public fork on upstream `rust-v0.122.0` and
republishes the latest Android Termux line as `0.122.0-termux`.

Fork-specific changes kept in this release:
- Android browser login still uses `termux-open-url`.
- Self-update and release links point to `DioNanos/codex-termux`.
- npm update/install flows target `@mmmbuto/codex-cli-termux@latest`.
- Voice and realtime audio remain disabled for the Termux package.
- Launcher hardening preserves helper re-exec behavior on Android.
- Android ELF `RUNPATH=$ORIGIN` hardening is included so bundled
  `libc++_shared.so` resolves on direct native invocation.
- Unsupported installation-id file locks are tolerated on Android so session
  bootstrap works on Termux.
- Code mode remains disabled on Android until the upstream runtime is usable in
  the Termux package.

Install or update:

```bash
npm install -g @mmmbuto/codex-cli-termux@latest
```

Known limitation: the packaged Termux build does not ship voice input or
realtime audio support.

## Upstream 0.122.0 highlights

- Standalone installs are more self-contained, and `codex app` improves
  Desktop handling on Windows and Intel Macs.
- The TUI adds `/side` conversations and queued slash or shell prompts while
  work is running.
- Plan Mode can start implementation in a fresh context and shows context usage
  before carrying a planning thread forward.
- Plugin workflows add tabbed browsing, inline enable/disable toggles,
  marketplace removal, and remote/cross-repo/local marketplace sources.
- Filesystem permissions add deny-read glob policies, managed deny-read
  requirements, stronger platform sandbox enforcement, and isolated
  `codex exec` runs.
- Tool discovery and image generation are enabled by default upstream, with
  higher-detail image handling.
